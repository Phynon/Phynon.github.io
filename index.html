发展历史

年代 成果
1943 神经元模型
1958 感知机
1960年之后 误差反向传播算法
1980 提出卷积神经网络
1983 提出 Hopfield 神经网络
1986 经典的误差反向传播算法
1989 发现三层神经网络的非线性拟合能力
1997 提出 LSTM 网络
1998 改进卷积神经网络
2006 提出深度置信网络
2006 提出 auto encoder

其中，监督学习的成果有卷积神经网络、深度置信网络和循环神经网络，非监督学习的成果有自动编码器 (auto encoder) 。

常见深度学习架构

Caffe 提供了在CPU和GPU上的快速卷积网络实现，使用 C++ 编写，灵活可移植，不易开发。

TensorFlow 基于张量的深度学习工具，是 Python 的域特定语言，带有丰富的算子和功能，但可读性有限。

Keras 可以使用多个后端，使用上更加简易，但相对更慢，不易开发调试。

PyTorch 基于 Torch 框架的开源 Python 机器学习库，支持自动求导，易开发调试，不易配置部署。

MXNet 兼具 Numpy 和 Keras 的特点，具有高性能和高实用性。

DGL 图神经网络工具包，有 PyTorch 和 MXNet 两种后端。

单周期MIPS处理器

编写代码

代码、综合实现后的文件和综合报告等文件打包在项目文件中。《数字逻辑与处理器基础》大作业中要求编写的仅有Control.v文件，顶层文件cpu.v中缺乏输出，综合不出正确的结果。因此，修改了cpu.v使之能够输出PC、$a0、$v0、$sp和$ra五个寄存器的值。

仿真和观察

时序分析

实现后生成时序分析报告，找出关键路径。WNS为21.911ns，在关键路径上的Arrival time为32.737ns。由此计算得到最高工作频率为30.546MHz。

接下来分析这条关键路径。先查看关键路径在整个原理图中的位置，再忽略其他元件，仅查看关键路径上的模块。关键路径的原理图如图所示。

从图中可以看出，关键路径从PC寄存器开始，经过指令存储器、寄存器堆和ALU后，进入数据存储器，然后又写回寄存器堆。这正是lw指令的数据通路，因此限制最高工作频率的关键路径就是lw的数据通路。

根据时序分析报告，关键路径内部，从ALU输出开始（第17.531ns），到数据存储器输出数据到Read_Data（第30.477ns），耗时12.946ns，为整个关键路径中耗时最长的部分。这段时间分为两部分，从第17.531ns到第24.153ns为寻址；第24.153ns之后，根据地址从内存中取出数据。因此可以说，数据存储器是最耗时的单元。其次耗时的单元为ALU。

在综合后实现前进行时序分析，得到的结果是beq产生zero信号的数据通路为关键路径。出现这种差异的原因可能是，综合时数据存储器也被当成是访存较快的寄存器处理，而实现后相当于把电路烧到FPGA里，数据存储器真正获得了RAM的性质。

逻辑资源分析

综合实现后观察device图，高亮部分的面积几乎占了2/3。查看项目总结当中的占用分析，发现小号最多的资源为IO，其次为触发器。

查看实现资源占用报告，发现以数据存储器占用的逻辑资源最多，其次为寄存器堆。这在一定程度上解释了存储器在单片机中占用面积较大的原因。

其余更详细的信息储存在vivado在项目文件夹中生成的综合和实现报告中。
